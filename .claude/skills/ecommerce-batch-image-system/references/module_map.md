# 模块映射说明

本文件用于把原始电商视觉系统提示词拆成可复用模块。注意：**这是模块化索引，不是压缩版替代品**。真正执行时，仍应以 `original_prompt_full.md` 为最高规则来源。

## 模块 A：总定位与总目标
来源：原文第 1-10 行
用途：定义技能不是简单写 prompt，而是输出完整商业视觉系统。
适用场景：任何正式输出前的总控约束。

## 模块 B：图生图一致性总约束
来源：工作前提部分
核心要求：
- 严格参考产品图
- 不改结构、不改标签、不改包装信息区
- 优化重点在构图、光影、场景、材质、镜头、层次、品牌氛围
适用场景：所有单图 prompt 都必须引用。

## 模块 C：职责范围
来源：职责部分
核心要求：
- 识别产品类型
- 分析卖点和人群
- 判断价格带与渠道
- 规划图组
- 为每张图写高密度 prompt
适用场景：生成整套方案时。

## 模块 D：低门槛交互
来源：交互原则部分
核心要求：
- 少提问
- 多补全
- 有足够信息就直接输出
适用场景：和用户交互时。

## 模块 E：自动补全引擎
来源：信息不足时的自动补全规则
补全部分包括：
- 产品属性
- 卖点类型
- 使用场景
- 视觉调性
- 价格带与视觉映射
- 渠道适配
- 图种优先级
适用场景：用户信息不全但需要直接推进时。

## 模块 F：六维分析模型
来源：分析模型部分
六类分析：
1. 产品识别分析
2. 卖点视觉化分析
3. 用户场景分析
4. 价格定位分析
5. 渠道分析
6. 风格统一分析
适用场景：输出整套方案的“需求简析”和“AI视觉方向建议”。

## 模块 G：单图 Prompt 结构骨架
来源：高密度商用 Prompt 章节
每张图至少要包含：
1. 图生图约束层
2. 商业任务层
3. 主体定义层
4. 镜头语言层
5. 构图层
6. 空间层
7. 材质层
8. 卖点视觉化层
9. 光影层
10. 氛围层
11. 版式留白层
12. 反向限制层

人物参与补充规则：
- 场景图可以加入人物手部、手持、半身人物、模特局部互动来增强真实代入感
- 人物是辅助叙事，不是主体；产品识别区、包装标签、核心结构必须清楚
- 人物加入后，仍需写清产品占比、焦点位置、人物与产品关系、禁止人物喧宾夺主
- 人物场景图必须绑定当前场景对应卖点，明确“人物动作如何证明卖点”“环境如何辅助卖点”“留白区如何承接卖点文案”
- 不同场景图必须分配不同卖点任务，避免多个场景图只重复同一种空泛生活方式表达
适用场景：所有单图 prompt 装配。

## 模块 H：图组规划引擎
来源：图组规划规则
图组类型：
- 主图组
- 卖点图组
- 场景图组
- 横版详情头图组
- 竖版详情图组
- 细节/材质图组
- 补充图组

扩展规则：
- 标准版默认规划到约 19-20 张
- 当用户说“拓展”“扩到 20 张”“再丰富一些”时，优先在场景图、竖版详情图、补充图中扩容，而不是只重复主图角度
- 场景图默认允许加入人物手部、手持、半身生活方式人物来增强代入感，但人物只能服务产品说明，不能喧宾夺主、不能遮挡产品识别区、不能让用户误以为主体变成人物大片
- 护肤/个护类场景图优先考虑：人物手持、梳妆台使用、上脸前准备、洗手台晨间护理、半身护肤生活方式场景
- 场景图在规划阶段就要分配不同卖点：有的负责补水、有的负责修护、有的负责清爽肤感、有的负责日常流程融入，不要都写成同一类氛围图
适用场景：根据模式生成批量任务。
## 模块 I：输出模式
来源：输出模式控制
模式对应关系：
- 精简版：约 9 条
- 标准版：约 19-20 条
- 完整版：20 条以上

画幅策略补充：
- 主图、商品卡、细节图、结构图通常优先正方形
- 卖点图、人物场景图、竖版详情图通常优先竖版
- Banner、横版头图、活动横幅通常优先横版
- 先根据渠道和信息承载做判断，再选画幅，不要默认全部 1:1
适用场景：控制批量生成数量与画幅策略。

## 模块 J：正式输出骨架
来源：输出格式要求
正式输出必须包含：
1. 需求简析
2. AI视觉方向建议
3. 推荐图种规划
4. 整套图生图提示词
5. 补充建议
适用场景：把静态方案交付给用户时。

## 模块 K：质量检查清单
来源：每条 Prompt 的质量标准 + 完整性补丁
检查项：
- 是否是长 prompt
- 是否有一致性约束
- 是否有商业用途
- 是否有镜头 / 构图 / 光线 / 空间 / 材质 / 氛围 / 留白 / 限制词
- 是否避免泛化和 AI 感
- 是否风格统一
- 人物场景图是否明确绑定卖点，是否能看出“场景为什么能证明这个卖点”
- `orientation` 与 `size` 是否匹配当前图种用途
适用场景：生图前自检。

## 模块 L：品类特殊规则
来源：不同品类的特殊规则
按品类选择：
- 护肤品 / 美妆 / 个护
- 家电 / 工具
- 食品 / 饮品
- 家居日用
- 母婴 / 宠物
- 服饰配件
适用场景：做具体品类增强时。

## 模块 M：风格推荐规则
来源：风格推荐规则
适用场景：用户没给风格词时自动推荐。

## 模块 N：重要注意事项与执行优先级
来源：最后三章
核心优先级：
1. 产品一致性与识别度
2. 商业用途与转化逻辑
3. 卖点视觉化与信息承载
4. 风格统一与品牌感
5. 高级感与美观度
适用场景：所有任务最终判定。

## 模块化装配建议

### 装配顺序
1. 先调模块 A + B + N 作为全局约束
2. 再调模块 C + E + F 完成需求分析
3. 再调模块 H + I 决定图组数量
4. 每张图使用模块 G 组装单图 prompt
5. 按品类需要附加模块 L
6. 用户没给风格时附加模块 M
7. 最后用模块 K 做质量复核

### 单图装配模板
每张图都应按以下逻辑生成：
- 全局一致性要求（B）
- 当前图的业务目标（J/H）
- 当前图绑定的核心卖点与场景证明逻辑（F/H/G）
- 产品与卖点分析结论（F/E）
- 单图结构骨架（G）
- 品类强化规则（L）
- 最终质量复核（K/N）
